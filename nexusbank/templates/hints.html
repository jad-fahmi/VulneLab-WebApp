Challenge Hints

NexusBank has several security vulnerabilities hidden throughout the application. Your goal is to find and exploit all of them. Each successful exploit reveals a flag. Use the hints below if you get stuck.

How This Works

There are 4 flags hidden in NexusBank. Find them all by exploiting the vulnerabilities.

  • Flags appear on the page after a successful exploit
  • Flag format: FLAG{vulnerability_name}
  • Try to find each vulnerability yourself before reading the hints
  • Available accounts: johndoe / password123, janesmith / letmein, bobwilson / qwerty
Challenge 01
Get In Without a Password
Easy

The login page has a critical vulnerability that allows you to bypass authentication entirely without knowing any valid password.

  • Hint 1: Think about how the login form sends your input to the server and how the server uses it to check the database.
  • Hint 2: What happens if your input contains special characters that the database interprets as part of its own language?
  • Hint 3: Try entering a single quote in the username field and observe what happens.
  • Hint 4: The username field is vulnerable. Try: ' OR '1'='1
Flag format: FLAG{sqli_nexusbank}
Challenge 02
Leave a Permanent Mark
Easy

The transfer system allows users to add comments to their transactions. These comments are displayed to everyone who views the transaction history.

  • Hint 1: What if the comment field accepts more than just plain text?
  • Hint 2: Try submitting a comment that contains HTML tags and see how the page renders it.
  • Hint 3: What if your comment contained a script tag? Would the browser execute it?
  • Hint 4: Try this as a transfer comment: <script>alert('XSS')</script>
Flag format: FLAG{xss_nexusbank}
Challenge 03
Look Where You Should Not
Medium

The profile page shows your personal information. But does it only show YOUR information?

  • Hint 1: Look carefully at the URL when you visit your profile page.
  • Hint 2: Is there a parameter in the URL that identifies which profile to show?
  • Hint 3: What happens if you change that parameter to a different number?
  • Hint 4: Try changing the id parameter in the URL to 2, 3 or 4.
Flag format: FLAG{idor_nexusbank}
Challenge 04
Find the Hidden Door
Hard

NexusBank has an admin panel that regular users are not supposed to access. There is no link to it anywhere on the site. Can you find it?

  • Hint 1: Admin panels on web applications are usually accessible via a URL. Think about what common admin URLs look like.
  • Hint 2: The application only checks if you are logged in, not what your role is.
  • Hint 3: Try navigating directly to common admin paths in the URL bar.
  • Hint 4: Try visiting /admin directly in your browser while logged in as a regular user.
Flag format: FLAG{bac_nexusbank}
